# 产品营销视频制作 - 使用说明

## 概述

这是一个基于 React + TypeScript + Vite 构建的产品营销视频制作 Web 应用。它允许你根据产品图片，自动生成多个营销角度的视频任务。

## 功能特点

- ✅ 产品图片上传和预览
- ✅ 5种预设营销角度（Premium Luxury, Smart Features, Lifestyle Daily, Performance Quality, Best Value）
- ✅ Hook-Body-CTA 结构的视频生成
- ✅ 支持语音解说和字幕
- ✅ 实时任务状态监控
- ✅ 响应式设计，支持移动端

## 使用步骤

### 步骤 1: 填写产品信息

- **产品名称**: 输入产品的英文名称（例如: Premium Wireless Headphones）
- **产品 ID**: 输入产品 ID（例如: WH-1000XM5）

### 步骤 2: 上传产品图片

- 点击上传区域，选择一张或多张产品图片
- 图片将显示在预览网格中
- 可以通过点击图片右上角的 × 按钮删除图片

### 步骤 3: 选择营销角度

从 5 种预设营销角度中选择一个或多个：

1. **Premium Luxury** - 高端奢华，强调工艺和优雅设计
2. **Smart Features** - 智能特性，突出创新功能
3. **Lifestyle Daily** - 日常生活，强调无缝集成
4. **Performance Quality** - 性能质量，强调可靠性和性能
5. **Best Value** - 超值优惠，强调性价比

### 步骤 4: 配置视频参数

- **视频时长**: 4-12 秒
- **启用语音解说**: 添加专业英文语音
- **启用字幕**: 添加匹配的英文字幕
- **Hook-Body-CTA 结构**: 使用吸引人-展示-呼吁三段式结构

### 步骤 5: 生成视频任务

点击 "🚀 生成视频任务" 按钮，系统将为每个选定的营销角度生成一个视频任务。

### 步骤 6: 监控任务状态

任务列表会显示所有任务的状态：
- **pending**: 待处理
- **generating**: 生成中（动画效果）
- **completed**: 已完成
- **failed**: 失败

## Hook-Body-CTA 结构说明

当启用 Hook-Body-CTA 结构时，视频将分为三个部分：

### Hook (0-20%)
- 吸引眼球的开场
- 产品戏剧性展示
- 文字覆盖: "Upgrade Your Life Today"
- 动态镜头运动

### Body (20%-80%)
- 特性展示
- 真实场景中的使用
- 多角度设计展示
- 平滑场景过渡
- 动态文字覆盖: "Premium Quality", "Smart Design", "Daily Essential"

### CTA (80%-100%)
- 强烈的行动呼吁
- 最终戏剧性镜头
- 文字覆盖: "Shop Now - Limited Time Offer"
- 紧迫感

## 技术架构

### 前端技术栈

- **React 19.2.0** - UI 框架
- **TypeScript** - 类型安全
- **Vite 7.3.1** - 构建工具和开发服务器
- **Tailwind CSS 4.2.1** - 样式框架
- **Zustand 5.0.11** - 状态管理
- **Axios** - HTTP 客户端

### API 接口

应用通过 HTTP 与 Mock Server 通信：

- `GET /` - 获取服务器状态
- `POST /api/tasks/push` - 提交新任务
- `GET /api/tasks` - 获取所有任务

### Prompt 生成

Prompt 生成器 (`lib/prompt-generator.ts`) 根据以下参数生成营销视频 prompt：

- 产品名称
- 营销角度
- 视频时长
- 语音/字幕选项
- Hook-Body-CTA 结构选项

## 故障排除

### Mock Server 未启动

如果页面显示 "Mock Server: 离线"，请确保：

1. Mock Server 在端口 3456 上运行
2. 使用 `curl http://localhost:3456/` 检查服务器状态

### 任务提交失败

如果任务提交失败，请检查：

1. Mock Server 是否正在运行
2. 所有必填字段是否已填写
3. 图片是否已上传
4. 至少选择了一个营销角度

### 样式显示异常

如果样式显示不正确，请尝试：

1. 清除浏览器缓存
2. 硬刷新页面（Ctrl+Shift+R 或 Cmd+Shift+R）
3. 重启开发服务器
